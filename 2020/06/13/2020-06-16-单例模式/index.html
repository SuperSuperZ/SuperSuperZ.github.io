单例模式

SuperSuperZ

2020-06-13

一、单例模式

1. 定义：

  • 保证一个类只产生一个实例，并提供一个访问它的全局访问点的一种创建型设计模式。

2. 使用场景：

  • 要求生成唯一序列号的环境；

  • 创建一个对象需要消耗的资源过多，如要访问IO和数据库等资源；

  • 需要定义大量的静态常量和静态方法（如工具类）的环境，可以采用单例模式（当然，也可以直接声明为static的方式）。比如工具类

  • 在整个项目中需要一个共享访问点或共享数据，例如一个Web页面上的计数器，可以不用把每次刷新都记录到数据库中，使用单例模式保持计数器的值，并确保是线程安全的；

3. 优点：

  • 减少维护对象的开销：创建、内存占用、回收

  • 避免对资源的多重占用（比如写文件操作）

4. 经典实现案例

  • Spring中，项目加载时bean自动创建,而后每次调用bean的时候由IOC容器注入。对于IOC容器，实现了Singleton模式。采用双重判断加锁的单例模式

  • JDK中，Runtime 类封装了Java运行时的环境。每一个java程序实际上都是启动了一个JVM进程，那么每个JVM进程都是对应这一个Runtime实例，此实例是由JVM为其实例化的。采用饿汉式单例模式。

5. 注意问题

  • 线程安全

  • 指令重排

  • 反射构造：私有构造函数抛异常 if (instance != null) throw new RuntimeException(“private constructor”);

  • 反序列化：添加readResolve方法，直接return instance;

6. 实现方式

  • 饿汉式

  • 懒汉式（线程不安全，严格来说不算是单例）

  • 懒汉式 (双重检测机制)

  • 懒汉式(绝对的线程安全)

  • 静态内部类方式

    • 和饿汉式有些类似，但是这个却是懒加载

  • 枚举类方式
