五、Activity 启动模式

  • Activity 启动 模式有 四种， 分别是： standard、singleTop、 singleTask、
    singleInstance，可以在 AndroidManifest.xml 文件中通过给标签指定
    android:launchMode 属性来选择启动模式。
  1. standard
    • 在没有指定情况下，所有 Activity 都默认使用 standard 启动模式。
    • 对于使用 standard 模式的 Activity，系统不会在乎这个 Activity 是否已经在栈中存在，每次启动都会创建该 Activity 的一个新的实例。
  2. singleTop
    • 在启动 Activity 的时候，如果栈顶已经是该 Activity，则认为可以直接使用，不会再创建新的 Activity 实例，否则则创建
  3. singleTask
    • 每次启动 Activity 时，系统首先会在栈中检查是否存在该 Activity 实例，如果发现已经存在该 Activity 实例，直接使用该实例，并把位于该 Activity 之上的所有 Activity 出栈
  4. singleInstance
    • 使用 singleInstance 模式的 Activity，会有一个单独的栈来管理这个 Activity，一般来说，这种模式是可以用来应用程序之间共享 Activity 实例，不管是哪个应用程序访问 Activity，都共用同一个栈。
    • 例如，应用主 Activity 是 A，A 启动了 B，B 是 singleInstance 模式的，然后活动 B 启动了 C，点击 Back 键，会从 C 跳转到活动 A，因为 A 和 C 在一个栈，而 B是另一个栈，在点击 Back 键，A 跳到了 B，再点击 Back 键，退出程序。

附、Activity生命周期

  1. 在 Android 系统中，采用 Activity 栈来维护和管理所有 Activity 组件，当
    一个新的 Activity 创建时，它就会放到栈顶，位于栈顶的 Activity 就处于运行
    状态。当再有一个新的 Activity 被创建后，会重新压人栈顶，而之前的 Activity
    则会在新的 Activity 底下，就像枪梭压入子弹一样。而且之前的 Activity 就会
    进入后台。当系统内存不足时，Android 系统会把位于栈底的 Activity 移除销
    毁，被系统回收。因此，Activity 的生命周期是指从创建到销毁的过程。

  2. 生命周期类型：活动、暂停、停止、销毁

  3. 活动生命周期中的回调方法

    • onCreate(): 当 Activity 第一次被创建时调用。是生命周期开始的第一个方
      法。

    • onStart():当 Activity 正在变为可见时调用。这个时候 Activity 已经可见
      了，但是还没有获得输入焦点，不能跟用户交互。还在应用程序后台

    • onResume():当 Activity 可以接收用户交互时调用。于OnStart相比已处于应用程序前台

    • 当 Activity 暂停时调用。在这里，可以用来保存数据，关闭动
      画和其它比较耗费 CPU 的操作；但是这里绝对不能做耗时的操作，因为如果当前
      Activity 要启动一个新的 Activity，新 Activity 会在当前 Activity 执行完毕
      onPause()之后才能进入可见状态。

    注意:在 Android3.0 之前，调用这个方法之后，Activity 可能会在系统内
    存紧张时被系统回收。

    • onStop():当 Activity 进入后台，并且完全看不见时调用。在调用该方
      法之后，系统在内存不够时可能回收该 Activity。

    • onDestroy()：当 Activity 被销毁时调用，该是 Activity 生命周期中最后
      一个调用方法。Activity 销毁原因：一般是程序中调用 Activity 的 finish()
      方法销毁，二是系统在内存不足时系统主动销毁 Activity，以用来节省空间，
      三是用户手动关闭Activity的界面。调用方法isFinishing()可以判断Activity
      是否正被销毁。

    • onRestart():在 Activity 又重新回到可视状态时调用。
      除了 Activity 生命周期回调方法以外，还有 onSaveInstanceState() 和
      onRestoreInstanceState()两个方法经常会被使用，用来保存和恢复 Activity
      的状态信息，例如用户在界面上输入的数据或选择的内容等

    • onSaveInstanceState()：不属于 Activity 的生命周期方法，在 Activity
      状态变化时，不一定会被触发。但是在应用遇到意外情况，比如：内存紧张，用户点击了 Home 键或者用户按下电源键关闭屏幕等，系统可能会销毁 Activity，
      但该 Activity 在未来某个时刻还会显示在屏幕，这时 onSaveInstanceState（）
      方法会被调用。需要强调的是系统回收 Activity，而不是用户手动销毁 Activity。

    • onRestoreInstanceState()： 这个方法调用的前提是，Activity 必须是被
      系统销毁了，在 Activity 被再次创建时它会在 onStart()方法之后被调用。

  4. 一般调用顺序onCreate()->onStart()->onResume()->onPause()->onStop()->onDestroy()
